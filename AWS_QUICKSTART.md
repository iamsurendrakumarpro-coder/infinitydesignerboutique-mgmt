# AWS Deployment Quick Start

**Total Time: 15-20 minutes** (plus 5-10 min RDS initialization wait)

## Prerequisites (10 minutes)

### A. Set Up IAM Service Account (IMPORTANT - Security Best Practice)

**DO NOT use root AWS credentials!** Create a service account first:

```bash
# 1. Sign in to AWS Console with ROOT credentials (one-time only)
# Go to: https://console.aws.amazon.com/

# 2. Run IAM bootstrap (creates runtime and provisioning users/profiles)
cd infinitydesignerboutique-mgmt
python scripts/aws_manage.py iam --use-default-credentials --region ap-south-1

# 3. Save the credentials displayed for both generated profiles
# ⚠️ You can only see them once! Save to secure location.
```

**See [IAM_SETUP.md](./IAM_SETUP.md) for detailed instructions & troubleshooting**

### B. Configure AWS CLI

```bash
# 1. Verify AWS CLI is installed
aws --version
# output: aws-cli/2.x.x ...

# 2. Verify the provisioning profile created by the IAM bootstrap
aws sts get-caller-identity --profile infinity-provisioner
# output: {"UserId": "...", "Account": "123456789012", "Arn": "arn:aws:iam::123456789012:user/infinity-provisioner-user"}

# 3. Install Python packages
pip install boto3 psycopg2-binary
```

## Deploy to AWS (3 Commands)

### Command 1: Create AWS Resources (2 minutes execution)

```bash
cd infinitydesignerboutique-mgmt

python scripts/aws_manage.py setup \
  --project-name infinity-boutique \
  --region ap-south-1 \
  --aws-profile infinity-provisioner \
  --output-env .env.aws
  
# This uses credentials from the generated infinity-provisioner profile
```

**Output:**
```
✓ RDS instance creation initiated: infinity-boutique-db
✓ S3 bucket created: infinity-boutique-files-ap-south-1
✓ Generated .env file: .env.aws
```

**What was created:**
- ✅ PostgreSQL database (20GB, db.t3.micro, free tier eligible)
- ✅ S3 bucket for file storage (versioning & security enabled)
- ✅ Security group allowing port 5432
- ✅ `.env.aws` configuration file

### Command 2: Wait for RDS (5-10 minutes) ☕

```bash
# Check creation progress (re-run every 2 minutes)
python scripts/aws_manage.py check \
  --project-name infinity-boutique \
  --region ap-south-1 \
  --aws-profile infinity-provisioner
  
# This uses credentials from the generated infinity-provisioner profile
```

**Expected output (after ~10 minutes):**
```
✓ RDS: available
✓ S3: available
✓ Security Group: available

PostgreSQL Connection String:
  postgresql://dbadmin:PASSWORD@infinity-boutique-db.xxxxx.ap-south-1.rds.amazonaws.com:5432/infinity_boutique
```

### Command 3: Deploy Application (1 minute)

```bash
# Copy configuration
cp .env.aws .env

# Create database tables
python scripts/init_db.py

# Start application
python app.py
```

**Output:**
```
 * Serving Flask app 'app'
 * Running on http://127.0.0.1:5000
Press CTRL+C to quit
```

Visit: http://localhost:5000 or deploy to production

## Verify Everything Works

```bash
# 1. Test database connection
psql -h $(grep POSTGRES_HOST .env | cut -d= -f2) \
     -U $(grep POSTGRES_USER .env | cut -d= -f2) \
     -d $(grep POSTGRES_DB .env | cut -d= -f2) \
     -c "SELECT version();"

# 2. Test S3 access
aws s3 ls s3://$(grep AWS_S3_BUCKET .env | cut -d= -f2)

# 3. Test application (in another terminal)
curl http://localhost:5000

# 4. Check logs
tail -f logs/app.log
```

## Next Steps

### For Production Deployment

1. **Scale up RDS**: Change `db.t3.micro` to `db.t3.small` or larger
2. **Enable Multi-AZ**: Automatic failover to another availability zone
3. **Set up backups**: Automated daily backups with 21-day retention
4. **SSL/TLS**: Use AWS Certificate Manager for HTTPS
5. **Load Balancer**: Add Application Load Balancer (ALB)
6. **CDN**: Use CloudFront for S3 static assets
7. **Monitoring**: Enable CloudWatch alarms for RDS and app

Run:
```bash
python scripts/aws_prod_setup.py --project-name infinity-boutique
```

### For Development

1. Keep current setup (safe and low-cost)
2. Seed predictable test data with `python scripts/reset_seed_and_run.py --seed-only`
3. Deploy to production when ready

## Troubleshooting

### RDS Still Creating After 15 Minutes?

```bash
# Check status
aws rds describe-db-instances \
  --db-instance-identifier infinity-boutique-db \
  --query 'DBInstances[0].DBInstanceStatus'

# If stuck, reboot
aws rds reboot-db-instance \
  --db-instance-identifier infinity-boutique-db
```

### Cannot Connect to Database?

```bash
# Get endpoint
aws rds describe-db-instances \
  --db-instance-identifier infinity-boutique-db \
  --query 'DBInstances[0].Endpoint.Address'

# Test with psql
psql -h <endpoint> -U dbadmin -d infinity_boutique

# Check security group allows your IP
aws ec2 describe-security-groups \
  --group-names infinity-boutique-rds-sg
```

### S3 Upload Fails?

```bash
# Verify bucket exists
aws s3 ls s3://$(grep AWS_S3_BUCKET .env | cut -d= -f2)

# Check IAM policy
aws iam list-policies --query 'Policies[?contains(PolicyName, `infinity-boutique`)]'
```

### Application Won't Start?

```bash
# Check Python packages
pip list | grep -E "boto3|psycopg2|flask"

# Test imports
python -c "import psycopg2, boto3; print('OK')"

# Check environment variables
env | grep -E "POSTGRES|AWS|FLASK"
```

| Issue | Solution |
|-------|----------|

| S3 access denied | Verify bucket name and IAM permissions |
